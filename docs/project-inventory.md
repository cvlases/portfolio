## Recommended Buckets

- `Code / Build`
- `Design / UX`
- `Research / Strategy`
- `Ethics / Planet / Public Interest`
- `Art / Visual`

Projects can live in more than one bucket, but choose one primary bucket for the archive.

## What To Gather For Each Project

- project title
- one-sentence hook
- year
- your role
- collaborators, if any
- tools used
- 2–6 strong images
- final outcome or impact
- link, if public
- quick note on what makes it special

## Fast Triage System

Use this when sorting:

- `featured` = strong story, strong visuals, relevant to target roles
- `archive` = solid work, but not top-tier for homepage attention
- `maybe` = interesting, but needs cleanup, context, or better assets

## Suggested First Pass

When you start filling this out, do it in this order:

1. list everything you can remember
2. mark each one `featured`, `archive`, or `maybe`
3. circle the ones with the best visuals
4. circle the ones most relevant to design engineering / creative tech / UX / research / ethics roles
5. choose the overlap as your homepage `featured` set
